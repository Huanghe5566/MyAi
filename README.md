# GD32C103 CAN Gateway & Controller Project

## 项目简介
本项目基于 **GD32C103** 系列微控制器开发，实现了一个高性能的 CAN 网关与控制器系统。主要功能包括多协议 CAN 消息解析、信号转换映射（由名爵 MG6 协议转换至 ME02 项目协议）、定时诊断报文注入以及基于按键交互的系统控制。

## 核心功能
- **CAN 协议转换**：解析源 CAN 网络（如 MG6 车速、横摆角速度、挡位信号），并将其重组映射为目标网络（ME02）的标准协议格式。
- **定时任务调度**：内置轻量级定时任务器，支持 10ms (车速/角速度)、100ms (挡位) 及 2000ms (诊断) 等多周期任务并发执行。
- **诊断报文注入**：自动周期性发送标准 UDS 诊断请求（ID: 0x700, 内容: `02 10 03 00...`）。
- **按键交互控制**：集成 `MultiButton` 状态机库，支持按键单击、双击等交互逻辑，用于触发特定硬件动作（如 LED 流水灯控制）。
- **状态监控**：通过串口（USART0）实时打印系统状态（车速、角速度、挡位等）并配合系统心跳灯（SYS_LED）进行运行指示。

## 硬件资源映射
- **MCU**: GD32C103VBT6
- **CAN 接口**:
  - CAN0: 连接整车/模拟网络 (标准 11 位 ID 模式)
  - CAN1: 支持 CANFD 接口模式
- **GPIO 控制**:
  - `SYS_LED`: 运行指示心跳灯
  - `LED1-LED8`: 状态指示及按键反馈灯组
  - `KEY1/KEY2`: 用户交互输入
- **定时器**:
  - `TIMER1`: 负责系统基础滴答与周期标志位更新
  - `TIMER2`: 专门驱动按键检测状态机 (1ms 扫描)

## 软件架构说明
- **工程环境**: Keil uVision 5
- **库文件**: GD32 标准固件库 (Standard Peripheral Library)
- **关键模块**:
  - `etk_can_custom.c`: 业务逻辑核心，包含 CAN 过滤规则与任务调度定义。
  - `can.c/h`: CAN 外设初始化与底层收发驱动。
  - `multi_button.c`: 第三方按键状态机库。
  - `main.c`: 系统初始化与主任务轮询入口。

## 配置与使用
1. **包含路径**: 确保 `.\USER` 目录已添加到编译器的 `Include Paths` 中。
2. **字符编码**: 项目文件已统一优化为 **UTF-8** 编码，解决中文注释在现代 IDE 中的乱码问题。
3. **编译烧录**: 直接使用 Keil 打开项目文件，编译生成 Hex 后通过 DAP 或 J-Link 烧录至开发板。
